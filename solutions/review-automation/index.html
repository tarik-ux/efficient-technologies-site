// workflow

How a fair review-request workflow operates

The system creates consistency. It does not decide which customers are allowed to leave a public review based on predicted sentiment.

  1. Receive a valid completion state

    An agreed job-complete or invoice-complete status starts the workflow.

  2. Apply one eligibility rule

    Operational exclusions such as duplicate, cancelled, test, or legally restricted records apply consistently—not by satisfaction score.

  3. Send the public review opportunity

    Every eligible customer receives the same clear path to the selected public review profile.

  4. Send a bounded reminder

    If the provider and consent rules permit it, one agreed reminder can be sent before the sequence stops.

  5. Record status

    Sent, delivered, clicked, replied, stopped, and failed states remain visible when the tools expose them.

  6. Route direct replies to a person

    Customer messages or service concerns create a human follow-up without removing the public review option.

Illustrative workflow, not a customer result: when an eligible completed job reaches the CRM, the system can send one review request, record delivery, stop after the approved reminder, and create a task if the customer replies directly.

// safeguards

What this system will not do

No review gating

It does not ask a private satisfaction question and show the public review link only to favorable respondents.

No positive-review incentive

It does not exchange discounts, gifts, entries, or benefits for a positive rating.

No fabricated review content

It does not write or submit a customer's review on the customer's behalf.

No endless reminders

The sequence has a documented limit, opt-out behavior, and human exception path.

// questions

Review automation FAQ

Can unhappy customers still leave a public review?

Yes. Every eligible customer receives the same public review opportunity. Direct service-recovery replies can route to a person without hiding that option.

Can we ask only customers who gave a high internal score?

No. That is review gating and is outside this service design.

How many reminders are sent?

The audit defines a small, consent-aware limit. The default design is one request and, when appropriate, one reminder.

Does this guarantee more or better reviews?

No. It guarantees the approved request workflow operates consistently. Whether a customer reviews and what they write remain customer decisions.
